# Лаборатория надёжности

Этот каталог собирает всё, что относится к устойчивости платформы: SLO, chaos-проверки, backup/restore, DR и операционные процедуры. Он работает поверх тех же сервисов и namespace, которые используются в релизном потоке.

## Что здесь лежит

- `docs/dr-plan.md` с целями восстановления и последовательностью failover
- `docs/backup-restore.md` с базовым сценарием Velero
- `docs/chaos-scenarios.md` и `chaos/*` для отказов pod, сети и узла
- `docs/slo-sli.md` с метриками и условиями rollback
- `docs/runbooks.md` и `docs/postmortem-example.md` для эксплуатации

## Как использовать этот слой

1. Сначала сервис должен быть развёрнут и наблюдаем.
2. После этого на тех же сигналах проверяются SLO и rollout-анализ.
3. Затем можно запускать chaos-сценарии и проверять runbook-и.
4. Для stateful-компонентов отдельно проверяются backup/restore и DR-процедуры.

## Рабочая зона

По умолчанию все практические проверки делаются в `demo-stage`, потому что там уже сходятся:

- rollout;
- метрики через `ServiceMonitor`;
- полный пользовательский путь через `api-gateway`, `orders-service` и `payments-service`;
- настройки, близкие к реальному тестовому окружению.

## Важное ограничение

Этот каталог не создаёт платформенные гарантии сам по себе. Он исходит из того, что платформа уже дала сервису probes, метрики, корректный namespace и совместимость с rollout.
